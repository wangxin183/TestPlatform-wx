# expected / 断言句式示例（产品文案仅作示意，以当前 App 为准）

生成用例时，关键可见文案必须用「」包裹。编译与自愈**不会**再根据无引号白名单猜测文案。

## 正向

- `右下角出现「追更」按钮`
- `会员条显示「开通会员」`
- `详情页可见「开始阅读」`

## 负向

- `会员条区域不出现「续费会员」`
- `「追更」按钮隐藏或不再可见`

## 禁止（无引号锚点）

- `追更按钮显示`（应写 `出现「追更」按钮`）
- `会员条不展示`（应写 `不出现「续费会员」` 或具体按钮文案）
- 单独用 `流畅` / `符合预期` / `与预期一致` 作为唯一判定
