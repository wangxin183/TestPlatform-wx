# 项目北极星与设计戒律

> 任何方案、补丁、配置抽取之前，先对照本文。忘记目标的「能跑」改动，一律视为失败设计。

## 本项目要做的事

打造面向 **ACN App UI 测试全流程**（需求分析 → 用例生成 → 编译 → 执行）的平台：

- **确定性高速路**：DSL、模块 Setup、页面锚点、路径缓存、token 组批、规则门禁 —— 便宜、可回放、可审计。
- **Harness 内 Agent 自主决策**：阻塞或歧义时，在工具/校验/预算围栏内诊断换招，直到**阶段目标**达成或记缺陷。
- **人不当日常 diagnoser**：人不修抖动；人只处理真缺陷与预算耗尽后的升级。

一句话：**用 Agent 能力平衡确定性与自主决策，向无人值守演进。**  
权威计划：`docs/superpowers/plans/2026-07-22-agent-engineering-self-heal.md`（StageAgentHarness）。

## 本项目要解决的场景问题

| 场景痛点 | 错误解法 | 正确方向 |
|----------|----------|----------|
| 人反复盯日志、改代码再跑 | 继续加 if/白名单/特例表 | HealLoop + role Agent + ledger |
| 用例能看不能跑 / 能跑不稳定 | 为单条用例写死文案或步骤 | 「」+ 状态合同 + 模块锚点 + Agent 工具 |
| 编译失败人看不懂 | 错误码→中文建议硬编码 map | `testcase.compile_advisor` 等 Agent 诊断 |
| App 文案/结构常变 | 产品 UI 文案白名单越积越大 | 规范写「」；动态内容靠状态/控件/Agent 观察 |
| 成本爆炸 | 整篇 Skill 反复塞 prompt | slim 指令 + 配置化组批；完整 Skill 只审计 |

## 分层：什么可以写死，什么不行

| 类型 | 可以 | 不可以 |
|------|------|--------|
| 工程调参 | `settings.yaml`（批大小、token、并发、超时） | 散落在业务 py 顶部的魔法数（除非瞬时常量） |
| 产品结构 | `acn_modules.yaml` 模块锚点、入口策略 | 在 TCG/Skill 硬塞「进 tab」步骤 |
| 策略词表 | `automation_lexicon.yaml`（主观/模糊词，相对稳定） | 每修一个 case 就往代码加一个特例字符串 |
| 产品 UI 文案 | 用例 expected「」；Skill `examples.md` 示意 | 编译/自愈白名单猜无引号文案 |
| 诊断/建议话术 | Agent + Skill | 错误码→suggestion 死表 |
| UI 状态展示 | badge / event→中文映射（必要映射） | 把业务决策逻辑塞进前端 map |

## 设计戒律（后续所有设计必须遵守）

1. **先问大局，再动刀子**  
   这个改动是在加强「确定性高速路」或「Agent 围栏内自主」，还是在给当前一个失败 case 打补丁？后者默认拒绝，除非补丁是可复用机制的最小步。

2. **禁止短视硬编码**  
   不为「解决当前问题而解决当前问题」。禁止：单用例特例、产品文案字典、错误码文案表、复制粘贴第二份词表。调试期捷径若不能晋升为机制，应删除而非配置化「升级」。

3. **能确定性则确定性，否则交给 Agent**  
   规则只守边界与安全（可测、可回放）；需要理解、改写、导航探索、诊断建议的，走 `agent_runtime` role，不在业务里写第二套「伪智能」。

4. **维护成本优先于当下省事**  
   模块会变多、文案会变、Agent 会换。设计问：三个月后谁改、改一处还是改十处？单源配置 / 单 Skill / 单 role，优于散落常量。

5. **改动前自检三问**  
   - 本项目要做的事是什么？（确定性 + Agent 自主）  
   - 本项目要解决的场景是什么？（无人值守测试流，人不修抖动）  
   - 这个方案三个月后是资产还是债？

## How to apply

- 新功能 / 修 bug：方案阶段写明落在「高速路」还是「Harness/Agent」，以及为何不是硬编码特例。  
- Code review：出现产品文案元组、错误码中文表、为单 TP 写的 if → 打回。  
- 与归档 [2026-07-23 硬化回顾](memory/archive/2026-07-23-hardening.md) 对照：白名单删除、compile_advisor、lexicon 即戒律落地样板。
